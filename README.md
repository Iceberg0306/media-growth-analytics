# 小红书内容增长数据分析（AI 辅助）

> 一句话：用 Claude Code + Python 构建了一套从数据采集到手策略输出的完整分析 Pipeline，驱动个人小红书账号 1 个月内实现 1.5w+ 赞藏、100+ 涨粉的冷启动成绩。

## 项目背景

2025 年 4 月底开始运营个人小红书宠物垂类账号（牡丹鹦鹉），从零启动。前两周凭直觉发内容，效果不稳定。5 月中旬开始用 Claude Code 做系统化数据分析，建立了"数据采集 → 特征工程 → 效果归因 → 策略迭代"的闭环。

## 我做了什么

### 1. 数据采集与指标定义（手动 + 脚本）
- 从小红书创作者后台导出 44 篇笔记的全量指标（曝光、点击率、点赞/收藏/评论/分享/涨粉、人均观看时长）
- 定义了**综合互动率 (IR)** = (点赞 + 收藏 + 评论) / 曝光 × 100%，作为核心效果指标
- 将标题文本手动清洗为可分析的结构化数据

### 2. 与 Claude Code 协作完成三轮递进式分析
- **Phase 1（探索性分析）**：体裁对比（图文 vs 视频）、TOP10 vs 倒数 10 对比，发现视频 IR 是图文的 1.76 倍
- **Phase 2（特征工程）**：构建了 4 个标题特征（省略号、字数、垂类标签词、微场景描述），对全量 44 篇做单变量 + 组合特征分析
- **Phase 3（严谨分析）**：扩展到 10+ 维度（发布时间段、发布星期、CTA 话术、情绪词、内容类型），**控制了体裁作为混淆变量**，按可信度分级输出结论

### 3. 关键发现（见 `reports/phase3_full.txt`）
| 结论 | 置信度 | 效应量 | 可执行策略 |
|------|--------|--------|-----------|
| 视频 IR 是图文的 1.76 倍 | 🟢 高 | +1.52pp | 优先发视频 |
| 标题 10-13 字最优 | 🟢 高 | +1.05pp | 控制字数区间 |
| 垂类标签词提升视频 IR | 🟢 高 | +2.38pp | 视频标题必含标签 |
| 深夜 21-24 点发布有优势 | 🟡 中 | +0.30pp | 优先深夜发 |
| 微场景描述有正向效果 | 🟡 中 | +0.57pp | 辅助优化 |

### 4. 策略落地与验证
- 根据分析结果调整内容方向：提高视频比例、标题控制在 10-13 字并嵌入垂类标签、集中在深夜发布
- **结果**：策略调整后的笔记平均 IR 提升约 40%（对比前 15 篇与后 29 篇）

## 技术栈

- **分析语言**：Python（openpyxl 数据读取、正则特征提取、统计分组）
- **AI 协作工具**：Claude Code（Prompt Engineering、迭代式分析引导）
- **分析方法**：单变量分组统计、效应量计算、混淆变量控制（分层分析）、可信度分级

## 面试准备

如果你想深入了解这个项目，以下是我建议的准备方向：

1. **为什么定义 IR 而不是用点赞率？** → 因为收藏和评论代表更深度的互动，对算法推荐权重更大
2. **为什么控制体裁？** → 视频 IR 天然高于图文，不控制的话会把体裁效应错误归因到其他特征上
3. **样本量只有 44 篇，结论可靠吗？** → 所以做了可信度分级，高置信度结论都验证了体裁分层后效应仍存在
4. **Claude Code 在这个项目里具体做了什么？** → 见我负责定义分析框架和 Prompt 方向，Claude Code 负责生成 Python 分析代码、发现我遗漏的维度、对分析结果做交叉验证
