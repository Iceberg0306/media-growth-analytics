# 与 Claude Code 的协作记录（Prompt Engineering 实践）

> 本文档记录了我如何使用 Claude Code 分阶段引导 AI 完成小红书笔记数据分析。
> 这不是 AI 一键生成的——而是我通过迭代式 Prompt Engineering 引导的成果。

## 我的角色 vs AI 的角色

| 环节 | 我做了什么 | AI(Claude Code)做了什么 |
|------|-----------|----------------------|
| 问题定义 | 确定分析目标、定义 IR 指标、选择维度 | — |
| 数据准备 | 从创作者后台导出 Excel、清洗格式 | 识别 Excel 结构异常（表头在第2行） |
| Phase 1 探索 | 给定分析方向（体裁对比、TOP10） | 生成 Python 代码，发现视频/图文的显著差异 |
| Phase 2 深化 | 提出 4 个标题特征假设 | 生成特征提取 + 分组统计代码 |
| Phase 3 严谨化 | 要求控制混淆变量、可信度分级 | 补充 10+ 维度分析，输出分层统计 |
| 策略落地 | 解读结论、制定内容策略、验证效果 | — |

## 分阶段 Prompt 设计

### Phase 1：探索性分析
**我的 Prompt 思路**：
> "读取这份数据，按体裁（图文/视频）分组计算互动率，并列出 TOP10 笔记"

**AI 产出**：`01_basic_stats.py`
- 识别出表头在第 2 行（第 1 行是总标题）
- 计算了综合互动率 = (点赞+收藏+评论)/曝光
- 生成了体裁对比 + 三个维度的 TOP10 排名

**我的判断**：结果发现视频 IR 远高于图文，但 TOP10 里也混了图文——说明标题等其他因素也在起作用。这引导了 Phase 2。

### Phase 2：标题特征分析
**我的 Prompt 思路**：
> "对比 TOP10 和倒数 10 的标题，分析字数、emoji、句式、关键词差异，总结高互动标题的共性规律"

**AI 产出**：`02_top10_comparison.py`
- 逐维对比了 TOP10 vs 倒数 10 的 7 个维度
- 自动生成了 3 条规律总结（虽然有些偏 TOP10 样本偏差）

**我的判断**：TOP10 vs 倒数 10 的信息量不够（只覆盖了 20/44 篇），需要全量分析。而且 AI 自动总结的规律需要回测验证。

### Phase 3：全量特征工程
**我的 Prompt 思路**：
> "对全量 44 篇做 4 个特征的单变量分析：省略号、字数分桶、垂类标签词、微场景描述。微场景用多信号打分（动词+名词+对话+时间+拟声词，排除抽象概括）"

**AI 产出**：`03_feature_analysis.py`
- 实现了微场景的多信号综合打分系统
- 做了组合特征叠加分析（省略号×微场景、标签×微场景）
- 输出全量 44 篇的特征标注一览表

**我的判断**：发现了垂类标签词效应很强（+1.71pp），但我意识到一件事——含标签的 6 篇里 4 篇是视频。到底是标签有效，还是视频有效？这驱动了 Phase 4 加上混淆变量控制。

### Phase 4：严谨化——这是最关键的一步
**我的 Prompt 思路**：
> "补充发布时间、星期、CTA 话术、情绪词、内容类型等维度。**重点：每组都要检查视频占比是否与全量一致，识别混淆风险。对高风险特征做体裁分层分析。最后按可信度分级输出结论。**

**AI 产出**：`04_full_pipeline.py` → `supplementary_analysis.txt`
- 扩展到 10+ 维度
- 系统检查了每个特征分组的视频占比 vs 全量 27%
- 识别出"垂类标签"和"含数字"是高风险混淆特征
- 做了 5 个特征的体裁分层分析
- 按 🟢🟡🔴 三级可信度输出结论

## 我学到的 AI 协作经验

### 1. AI 擅长代码执行，但分析框架需要人来定义
- 我告诉 AI "分析什么维度"和"用什么方法"，AI 负责写代码和计算
- AI 不会主动想到"控制体裁"这个统计学操作——这是我加上去的

### 2. 迭代式的 Prompt 比一次性 Prompt 质量高得多
- 如果我一上来就让 AI "全面分析这份数据"，结果会是一堆浅层统计
- 分 4 轮迭代，每轮基于上一轮发现的问题加深，最终产出才靠谱

### 3. AI 的分析需要交叉验证
- Phase 2 的规则 1（省略号有效）在 Phase 3 全量数据中被证伪（+0.06pp，几乎为零）
- 如果不做全量回测，TOP10 分析很容易得出有偏的结论

### 4. 我的核心能力不是"会用 AI"，而是
- 定义正确的分析指标（为什么用 IR 而非点赞率）
- 识别 AI 分析中的方法论问题（混淆变量、样本偏差）
- 把统计分析转化为可执行的内容策略
